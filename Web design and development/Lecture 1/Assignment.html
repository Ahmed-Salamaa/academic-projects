1. Framework vs Library

Aspect

Framework

Library

Definition

A framework is a complete structure or skeleton that dictates how your application should be built.

A library is a collection of reusable functions or classes you can call when needed.

Control

The framework calls your code (“Inversion of Control”).

You call the library’s code directly.

Example (Web Design)

Angular, React (technically a library but used as a framework), Vue.js, Next.js

jQuery, Lodash, Axios

Usage in Web Design

Defines the whole application’s flow, routing, and structure (frontend or backend).

Provides specific features like animations, DOM manipulation, or HTTP requests.

2. Compiler vs Preprocessor

Aspect

Compiler

Preprocessor

Definition

Translates source code into machine code or another language (e.g., TypeScript → JavaScript).

Processes code before it’s compiled — usually for macros, includes, or variable replacements.

When it runs

After preprocessing.

Before compilation.

Example (Web Design)

Babel or TypeScript compiler turns TypeScript or JSX into JavaScript.

SASS or LESS preprocessors convert special syntax into CSS.

Output

Executable or browser-readable code.

Source code ready for compilation or direct browser use.

3. TypeScript vs JavaScript

Aspect

TypeScript

JavaScript

Definition

A superset of JavaScript that adds static typing and compile-time error checking.

A dynamic scripting language interpreted by browsers.

Type System

Strongly typed (you define data types).

Loosely typed (types are decided at runtime).

Compilation

Needs to be compiled (using the TypeScript compiler).

Runs directly in browsers.

Error Detection

Errors found during development (before running).

Errors found at runtime.

Use in Web Design

Used in large, scalable applications (Angular uses it).

Used everywhere — all browsers understand JavaScript directly.
